hamsterdb::env::~env ( )
inline

Destructor - automatically closes the Environment, if necessary.

!! Any exception is silently discarded. Use of the destructor to close environments is therefore not recommended, because there are valid reasons why an Exception can be thrown (i.e. not all Databases were closed or not all Transactions were committed/aborted).

An assert() was added to catch this condition in debug builds.
